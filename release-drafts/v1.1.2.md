## v1.1.2 — Model-tier drift fix

Tiny patch release correcting two drifted agent defaults that have been shipping since v1.1.0.

### Fixed

- **`fsm-executor` default model.** Was `sonnet`; corrected to `haiku`. The `Model Tier Defaults` table in `CLAUDE.md` has always listed executor as `haiku` ("all executor tasks are atomized single-step. Speed + 529 headroom"), but `plugins/fsm-workflow/agents/fsm-executor.md` has been shipping `sonnet` for three releases.
- **`fsm-integrator` default model.** Was `opus`; corrected to `sonnet`. `CLAUDE.md` lists integrator as `sonnet` with dispatcher-level Opus escalation available on demand. Frontmatter was hardcoding `opus`, defeating the escalation tier.
- **CHANGELOG footer compare links** for `[1.1.1]` and `[Unreleased]` were never added during the v1.1.1 bump. Added alongside the `[1.1.2]` entry.

### Why it matters

Marketplace installs and direct clones pick up the agent frontmatter verbatim — a drifted `model:` line silently overrides the intended routing tier and defeats the rate-limit-aware planning encoded in `CLAUDE.md`. `fsm-executor` on Sonnet spends 4–5× the tokens of the intended Haiku baseline; `fsm-integrator` on Opus burns the 5-hour budget and creates unnecessary 529 pressure. Both are now corrected.

### No breaking changes

Pure frontmatter edit. No CLI, hook, or task-file contract changes. Re-run `install.sh` (or re-install the plugin) to pick up the new defaults.
